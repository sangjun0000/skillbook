# SaaS 전략 워크플로우

> 유닛 이코노믹스가 검증되지 않은 성장 전략은 절대 제안하지 않는다.

## 게이트 (반드시 먼저)

전략을 제안하기 전에 반드시 완료해야 하는 선행 조건.
통과하지 않으면 다음 단계로 진행하지 않는다.

- [ ] 현재 LTV/CAC 비율을 계산했는가 (3:1 미만이면 성장 전략 전에 효율성 개선 권고)
- [ ] 사용자당 월간 LLM/API 비용을 산출하여 Gross Margin 60% 이상 확보 가능한지 확인했는가
- [ ] 현재 단계를 분류했는가 — PMF 탐색(0→$10K MRR) / 초기 성장($10K→$100K) / 스케일링($100K+)
- [ ] 가입 코호트별 월간 리텐션 커브를 확인했는가 (없으면 수집 방법부터 제안)

## 규칙 (항상 따라야 함)

1. **단계 적합성 강제**: PMF 단계에서 유료 광고, Enterprise 기능, 글로벌 확장을 절대 제안하지 마라. 단계별 우선순위를 엄격히 준수한다.
2. **AI 비용 반드시 모델링**: 프라이싱 제안 시 사용자당 API 호출 비용을 계산하고, Gross Margin 목표(최소 60%)를 명시한다.
3. **Vanity Metrics 거부**: 가입자 수, 페이지뷰, 다운로드 수만 언급하는 경우 실질 지표(활성 사용자, 유료 전환율, NRR)로 전환하도록 즉시 교정한다.
4. **코호트 분석 의무화**: 리텐션을 논할 때 반드시 가입 시점별 코호트 분석을 수행한다. 전체 평균 리텐션만 제시하지 마라.
5. **프라이싱은 가치 기반**: 비용 기반 프라이싱(원가 + 마진)을 절대 제안하지 마라. 고객이 얻는 가치를 정량화하고 그 10-20%를 가격으로 설정한다.
6. **NRR 100% 기준선**: Expansion Revenue 전략 없이 신규 고객 획득만 강조하지 마라. NRR이 100% 미만이면 리텐션/확장 전략이 우선이다.

## 완료 체크리스트 (통과해야 완료)

작업 완료 전 반드시 확인. 하나라도 실패하면 완료 아님.

- [ ] LTV/CAC, Gross Margin, Payback Period 세 지표가 모두 계산되었는가
- [ ] 제안된 전략이 현재 성장 단계에 적합한가
- [ ] 프라이싱에 AI API 비용 헤징 방안이 포함되었는가
- [ ] 코호트 리텐션 데이터 기반으로 리텐션 분석이 이루어졌는가
- [ ] 즉시 실행 가능한 Action Item이 단계별(이번 주/이번 달/분기)로 정의되었는가
