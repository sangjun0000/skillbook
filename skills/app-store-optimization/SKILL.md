# 앱스토어 최적화(ASO) 워크플로우

> 글자 수를 채우지 않은 메타데이터와 순서가 틀린 스크린샷은 존재하지 않는다.

## 게이트 (반드시 먼저)

메타데이터나 에셋을 작성하기 전에 반드시 완료해야 하는 선행 조건.

- [ ] 타겟 키워드 목록이 볼륨/난이도/관련성 3축으로 평가되어 있는가
- [ ] 상위 3개 경쟁 앱의 앱 이름과 키워드 필드가 분석되어 있는가
- [ ] 배포 대상 플랫폼(App Store / Play Store / 양쪽)이 확정되어 있는가

## 규칙 (항상 따라야 함)

1. **App Store 글자 수 준수**: 앱 이름 30자, 부제목 30자, 키워드 필드 100자를 반드시 꽉 채운다. 남은 글자 수는 낭비된 노출 기회다
2. **키워드 중복 금지**: 앱 이름에 이미 포함된 단어를 키워드 필드에 다시 넣지 않는다. App Store 알고리즘이 중복을 무시하므로 신규 키워드로 채운다
3. **스크린샷 첫 장 = Hero Shot**: 첫 번째 스크린샷은 반드시 핵심 가치 제안 한 문장을 텍스트 오버레이로 포함한다. UI만 캡처한 스크린샷을 첫 장에 놓지 않는다
4. **스크린샷 순서 고정**: 1장 핵심 가치 → 2장 주요 기능 → 3장 차별화 포인트 → 4장 사회적 증거 → 5장 CTA 순서를 따른다
5. **리뷰 요청 4가지 조건 동시 충족**: 핵심 기능 성공 사용 + 세션 5회 이상 + 설치 후 7일 이상 + 마지막 요청 후 90일 이상, 모두 충족할 때만 리뷰 팝업을 트리거한다
6. **부정 피드백 인앱 유도**: "불만족" 응답은 스토어 리뷰 대신 인앱 피드백 채널로 분기한다. 절대 불만족 사용자를 스토어 리뷰로 보내지 않는다
7. **Play Store 설명 키워드 밀도**: 전체 설명 내 핵심 키워드 밀도를 2~5%로 유지한다. 동일 키워드를 문장당 1회 이상 반복하지 않는다
8. **절대 지역화 없이 글로벌 출시하지 마라**: 타겟 시장의 언어로 메타데이터를 개별 최적화한다. 영어 메타데이터를 번역만 하는 것은 현지화가 아니다

## 완료 체크리스트 (통과해야 완료)

- [ ] App Store: 앱 이름 30자 이하, 부제목 30자 이하, 키워드 100자 이하 — 각 한도에 근접하게 채워져 있는가
- [ ] 키워드 필드에 앱 이름/부제목에 이미 포함된 단어가 없는가
- [ ] 첫 번째 스크린샷에 텍스트 오버레이가 포함되어 있는가
- [ ] 스크린샷이 Hero Shot → 기능 → 차별화 → 증거 → CTA 순서인가
- [ ] 리뷰 요청 트리거가 4가지 조건을 모두 확인하는 로직으로 구현되어 있는가
- [ ] 불만족 응답이 인앱 피드백으로 분기되는가
- [ ] 타겟 시장별 별도 메타데이터가 존재하는가
