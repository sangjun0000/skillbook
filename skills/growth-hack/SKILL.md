# 그로스 해킹 워크플로우

> 측정 없이 실험하지 않고, 가설 없이 기능을 만들지 않는다.

## 게이트 (반드시 먼저)

코드나 전략을 작성하기 전에 반드시 완료해야 하는 선행 조건.

- [ ] North Star Metric이 단일 지표로 정의되어 있는가 (DAU, MRR, "X행동 완료 수" 등)
- [ ] AARRR 각 단계의 현재 전환율 수치가 확보되어 있는가
- [ ] Aha Moment가 데이터(리텐션 상관관계)로 검증되어 있는가 — 없으면 정의부터 수행

## 규칙 (항상 따라야 함)

1. **가설 선행**: 실험을 시작하기 전에 반드시 "만약 X를 하면, Y 지표가 Z% 개선될 것이다" 형식으로 가설을 명시한다
2. **측정 체계 선행**: 기능 구현 전에 이벤트 추적 코드가 존재하는지 확인한다. 없으면 측정 코드를 먼저 작성한다
3. **North Star 정렬**: 제안하는 모든 그로스 실험은 North Star Metric과의 연결을 명시한다. 연결이 불명확하면 제안하지 않는다
4. **통계적 유의성 요구**: 실험 결과를 판단할 때 최소 표본 크기(n≥1,000)와 유의 수준(p<0.05)을 만족하지 않으면 "데이터 부족"으로 판정한다
5. **리텐션 우선**: 신규 유입 채널 확대를 제안하기 전에 반드시 현재 D7 리텐션이 10% 이상인지 확인한다. 미달이면 리텐션 개선을 먼저 권고한다
6. **절대 Vanity Metric 추적하지 마라**: 가입자 수, 다운로드 수를 목표 지표로 설정하지 않는다. Activation Rate, D7 Retention, Revenue Conversion을 쓴다
7. **절대 강제 바이럴을 설계하지 마라**: 핵심 기능 잠금 + 초대 강요 방식은 제안하지 않는다. 가치 발생 순간에 자연스럽게 공유 루프를 배치한다

## 완료 체크리스트 (통과해야 완료)

- [ ] 모든 실험 항목에 가설 문장이 작성되어 있는가
- [ ] 각 실험의 성공 지표와 최소 표본 크기가 사전 정의되어 있는가
- [ ] 제안된 모든 전략이 North Star Metric 개선으로 귀결되는가
- [ ] 리텐션 현황(D1/D7/D30)이 진단에 포함되어 있는가
- [ ] Aha Moment 도달 경로(단계 수, 소요 시간)가 명시되어 있는가
- [ ] 그로스 실험 백로그가 임팩트/난이도 기준으로 우선순위가 지정되어 있는가
