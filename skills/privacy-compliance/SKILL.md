# 개인정보보호 컴플라이언스 워크플로우

> 개인정보는 수집 근거 없이 기본값으로 수집하지 않는다. 처리 목적이 명확하지 않으면 코드를 작성하기 전에 먼저 법적 근거를 확정한다.

## 게이트 (반드시 먼저)

코드를 작성하기 전에 반드시 완료해야 하는 선행 조건.
통과하지 않으면 다음 단계로 진행하지 않는다.

- [ ] 수집·처리하는 개인정보 항목 전체 목록화 완료 (이름, 이메일, IP 포함)
- [ ] 각 항목별 GDPR Article 6 / 개인정보보호법 제15조 처리 근거 확정 (동의 / 계약 / 법적 의무 / 정당한 이익 등)
- [ ] 민감정보(건강, 생체, 정치적 견해 등) 포함 여부 확인 — 포함 시 GDPR Art. 9 / 개보법 제23조 특별 근거 필요
- [ ] 제3자 제공 또는 국외 이전 여부 파악
- [ ] 기존 개인정보처리방침(Privacy Policy) 문서 위치 확인 및 최신 여부 검토
- [ ] DPO(개인정보보호책임자) 또는 담당자 지정 여부 확인

## 규칙 (항상 따라야 함)

1. **처리 근거 없이 수집 금지 (GDPR Art. 6 / 개보법 제15조)**: 개인정보를 수집·처리하기 전에 반드시 6가지 합법 근거 중 하나를 명시한다 — 동의(Consent), 계약 이행(Contract), 법적 의무(Legal Obligation), 생명 보호(Vital Interests), 공공 이익(Public Task), 정당한 이익(Legitimate Interests). 근거 없는 처리는 위반이며 코드 작성을 중단한다.

2. **데이터 최소화 원칙 강제 (GDPR Art. 5(1)(c))**: 처리 목적에 필요한 최소한의 데이터만 수집한다. 선택 입력 필드가 있다면 수집하지 않을 경우 서비스 이용에 지장이 없어야 한다. 마케팅 프로파일링을 위한 추가 수집은 별도 동의를 받아야 하며 기본값은 OFF다.

3. **DPIA 의무 조건 검증 (GDPR Art. 35)**: 다음 중 하나라도 해당하면 DPIA(개인정보 영향평가)를 실시해야 한다 — 체계적 자동화 프로파일링, 대규모 민감정보 처리(EU 인구 1% 초과 추정), 공공장소 대규모 모니터링, 신규 기술을 이용한 처리. 조건 해당 여부를 게이트에서 판단하고, 해당 시 DPIA 없이 배포하지 않는다.

4. **침해 시 72시간 통지 의무 (GDPR Art. 33 / 개보법 제34조)**: 개인정보 침해 사고 발생 시 감독기관(GDPR: DPA, 국내: 개인정보보호위원회)에 72시간 내 신고해야 한다. 정보주체에게 영향이 있는 경우 지체 없이 개인에게도 통지(Art. 34). 이를 위해 침해 감지 → 내부 에스컬레이션 → 통지 절차를 시스템에 반드시 구현한다.

5. **쿠키 동의 선처리 의무 (GDPR ePrivacy Directive / 국내 정보통신망법 제22조의2)**: 분석·마케팅 쿠키는 사용자가 동의하기 전까지 설정하지 않는다. 필수 쿠키(세션, 보안)만 사전 동의 없이 허용된다. 동의 배너는 '모두 허용'과 '거부'를 동등하게 노출해야 하며, '모두 허용'이 기본 선택 상태여서는 안 된다.

6. **국외 이전 안전장치 의무 (GDPR Chapter V / 개보법 제28조의8)**: EU/EEA 외부로 데이터를 이전할 때는 반드시 하나를 적용한다 — 적정성 결정국(Adequacy Decision), 표준계약조항(SCCs, 2021년 개정판 사용), 구속력 있는 기업 규칙(BCRs). 국내법 기준으로는 정보주체 동의 또는 개인정보 국외이전 표준계약 체결이 필요하다.

7. **정보주체 권리 이행 체계 구축 (GDPR Art. 15-22 / 개보법 제35-39조)**: 열람(Access), 정정(Rectification), 삭제(Erasure / Right to be Forgotten), 처리 제한(Restriction), 이동(Portability), 반대(Objection) 요청을 처리할 API 또는 절차가 반드시 존재해야 한다. GDPR 기준 응답 기한은 30일(최대 90일 연장 가능), 국내법 기준 10일 이내. 삭제 요청 시 모든 연계 시스템(백업, 로그, 서드파티)에서도 삭제되어야 한다.

8. **보존 기간 명시 및 자동 삭제 (GDPR Art. 5(1)(e) / 개보법 제21조)**: 모든 개인정보에 보존 기간을 명시하고, 기간 도래 시 자동 삭제 또는 익명화 처리하는 스케줄러를 구현한다. "필요한 기간 동안 보존"은 허용되지 않는 표현이다 — 구체적 기간(예: 회원 탈퇴 후 30일, 거래 완료 후 5년)을 정의해야 한다.

9. **Privacy by Design 기본값 적용 (GDPR Art. 25)**: 시스템 기본 설정은 항상 가장 프라이버시 보호적인 옵션이어야 한다. 마케팅 수신, 위치 추적, 행동 분석은 기본값 OFF로 설정하고 사용자가 명시적으로 활성화해야 한다. 새 기능 설계 시 개인정보 최소화를 초기 요구사항으로 포함한다.

10. **DPO 지정 의무 확인 (GDPR Art. 37)**: 다음 중 하나라도 해당하면 DPO를 지정해야 한다 — 공공기관, 정보주체를 대규모로 정기적·체계적으로 모니터링하는 경우, 대규모 민감정보 처리. 국내법은 개인정보보호책임자(CPO) 지정이 원칙적으로 모든 사업자에게 의무다(개보법 제31조).

## 완료 체크리스트 (통과해야 완료)

작업 완료 전 반드시 확인. 하나라도 실패하면 완료 아님.

- [ ] 모든 개인정보 수집 항목에 GDPR Art. 6 또는 개보법 제15조 처리 근거가 코드/문서에 명시되어 있는가
- [ ] 민감정보(건강, 생체, 종교 등) 처리 시 GDPR Art. 9 / 개보법 제23조 별도 근거 및 동의가 적용되어 있는가
- [ ] DPIA 필요 조건 검토가 완료되었고, 해당 시 DPIA 문서가 존재하는가
- [ ] 분석·마케팅 쿠키가 사용자 동의 전에는 설정되지 않으며, 쿠키 배너가 '거부' 옵션을 동등하게 제공하는가
- [ ] 국외 데이터 이전 시 SCCs / 적정성 결정 / BCRs 중 하나가 적용되어 있는가
- [ ] 열람·정정·삭제·이동 권리 요청을 처리하는 엔드포인트 또는 절차가 구현되어 있는가 (응답 기한 30일 / 국내 10일)
- [ ] 모든 개인정보 유형에 구체적인 보존 기간이 정의되고 자동 삭제 로직이 존재하는가
- [ ] 마케팅·분석·위치 추적 기능의 기본값이 OFF(Privacy by Design)인가
- [ ] 72시간 침해 통지를 위한 내부 에스컬레이션 절차 또는 담당자가 지정되어 있는가
- [ ] 개인정보처리방침이 현재 처리 활동을 정확히 반영하고 있으며 접근 가능한 위치에 게시되어 있는가
