# 브랜드 아이덴티티 워크플로우

> 대비 검증을 통과하지 못한 색상과 예시 없는 톤앤매너 가이드는 완성이 아니다.

## 게이트 (반드시 먼저)

색상, 타이포, 톤앤매너를 정의하기 전에 반드시 완료해야 하는 선행 조건.

- [ ] 브랜드 미션, 핵심 가치(3~5개), 타겟 사용자가 정의되어 있는가
- [ ] 경쟁사 3개 이상의 색상/타이포/톤을 비교 분석했는가 — 차별화 포인트 식별을 위해
- [ ] 기존 브랜드 에셋(로고, 색상, 폰트)이 있다면 현황 감사가 완료되어 있는가

## 규칙 (항상 따라야 함)

1. **WCAG AA 대비 검증 필수**: 모든 브랜드 색상을 정의할 때 일반 텍스트 4.5:1, 대형 텍스트 3:1 대비비를 계산한다. 수치 없이 색상을 확정하지 않는다
2. **다크 모드 대비 동시 검증**: 라이트 모드 팔레트를 정의할 때 다크 모드 버전의 대비비도 함께 검증한다. 하나만 통과한 색상은 채택하지 않는다
3. **터치포인트 일관성 점검 의무화**: 웹, 앱, 이메일, SNS 4개 채널에서 동일 색상/폰트/톤이 적용되는지 명시적으로 확인한다. 채널이 누락되면 점검 완료로 보지 않는다
4. **Voice/Tone은 예시 쌍으로 문서화**: 톤앤매너 특성을 정의할 때 "이렇게 써라 / 이렇게 쓰지 마라" 예시를 반드시 쌍으로 작성한다. 설명만 있고 예시가 없는 항목은 미완성이다
5. **절대 전략 없이 색상을 선택하지 마라**: 브랜드 가치와 성격 포지셔닝을 먼저 정의한 후 색상/폰트를 선정한다. "예쁜 색"으로 시작하는 팔레트는 허용하지 않는다
6. **절대 7개 이상 브랜드 색상을 정의하지 마라**: Primary 1개 + Secondary 최대 2개 + Neutral + Semantic(성공/경고/오류)으로 제한한다. 초과 시 통합 또는 제거한다
7. **로고 변형은 컴포넌트로만**: 허용된 로고 변형(full/symbol/wordmark, light/dark/mono)을 컴포넌트로 구현하고, 임의 색상/비율 변경을 코드 수준에서 차단한다

## 완료 체크리스트 (통과해야 완료)

- [ ] 모든 브랜드 색상의 WCAG AA 대비비(4.5:1)가 수치로 기록되어 있는가
- [ ] 라이트 모드와 다크 모드 양쪽의 대비 검증이 완료되어 있는가
- [ ] 웹/앱/이메일/SNS 4개 터치포인트에서 색상/폰트/톤 일관성이 확인되었는가
- [ ] Voice/Tone 가이드에 각 특성마다 "해야 할 예시 / 하지 말아야 할 예시" 쌍이 있는가
- [ ] 브랜드 색상이 Primary 1개 + Secondary 최대 2개 + Neutral + Semantic 이내인가
- [ ] 로고 변형이 허용된 variant 외 임의 수정이 불가능한 구조로 구현되어 있는가
